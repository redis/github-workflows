## Creating New Specs

1. Name: `feature-name.md`
2. Include: goal, user stories, acceptance criteria, constraints
3. Add entry to this index
4. Create execution plan in `docs/exec-plans/active/` when work begins

## Linking to Implementation

Each spec should reference:
- Related design docs in `docs/design-docs/`
- Execution plan in `docs/exec-plans/`
- Relevant code locations
